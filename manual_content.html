Activeチャプター 名簿・割り振りシステム ご利用マニュアル

本システムは、Googleスプレッドシート上で 「ビジターリストの作成」「ブレイクアウトルームの割り振り」「案内メールの一括送信」 を半自動化し、定例会の準備にかかる作業時間を大幅に削減するための専用ツールです。

【重要】初回実行時のGoogle認証について

スプレッドシートのメニューから初めてシステム（スクリプト）を起動した際、Googleから 「承認が必要です」 というメッセージが表示されます。これは、システムがあなたの代わりにPDFを作成したりメールを送信したりするための許可を求めているためです。

以下の手順で許可を行ってください。

  1. 「承認が必要です」のポップアップで 「続行」 をクリックします。
  2. アカウントの選択画面で、ご自身の Googleアカウント をクリックします。
  3. 「Google はこのアプリを確認していません」という警告画面が出たら、左下の 「詳細」 をクリックします。
  4. 一番下に表示される 「〇〇（安全ではないページ）に移動」 をクリックします。
  5. 最後に 「許可」 をクリックします。

この操作は初回のみです。次回以降は表示されません。

本システムが要求する権限（スコープ）

権限 用途
Google スプレッドシートの読み書き 名簿データの読み書き、シートの作成
Google ドライブへのアクセス PDF ファイルの作成・保存・共有設定
Gmail の送信 ビジター向け案内メールの一括送信
外部サービスへの接続 (UrlFetch) Gemini API の呼び出し（AI自動割り振り機能）
Google ドキュメントへのアクセス メンバーリストPDFのOCR処理（Google Docs変換による文字抽出）

1. 事前準備（データの入手と確認）

定例会の準備を始める前に、以下のデータを手元に用意してください。

1-1. ビジターリストのCSVファイル

「Spreading」にログインし、当該の定例会を選択して「CSVダウンロード」からパソコンに保存してください。

補足: SpreadingからダウンロードされるCSVは Shift_JIS（SJIS） エンコーディングです。本システムはShift_JISを自動的に読み取るよう設計されているため、エンコーディングの変換は不要です。ダウンロードしたファイルをそのままお使いください。

1-2. メンバーリストのPDFファイル

最新のメンバーリストPDFを、専用インターフェースからダウンロードしておいてください。

1-3.「紹介したい（つなげたい）メンバー」の確認

重要: Spreadingの仕様上、「紹介したいメンバー（つなげたいメンバー）」の情報はCSVファイルに書き出されません。

そのため、お手数ですがSpreadingの画面上で各ビジターの詳細情報を個別に開き、誰とつなげたいのかを確認・メモしておく（または後述の割り振り画面でコピペする）必要があります。

2. 初期設定（初回・変更時のみ実行）

メニューバー右端の 「名簿システム」 から、環境の土台となる設定を行います。

2-1. メンバーリスト(OCR)の更新

メニュー: 名簿システム > メンバーリスト(OCR)の更新

ダウンロードした「メンバーリストのPDF」をアップロードします。

  • アップロードされたPDFはGoogle ドキュメントに変換され、OCRでテキストが抽出されます。
  • 抽出された番号と氏名が「メンバーリスト」シートに自動書き出しされます。
  • 処理後、一時的に作成されたGoogle ドキュメントは自動削除されます。
  • 結果確認: 処理完了後、「メンバーリスト」シートを開いて抽出結果を確認してください。名前の読み取りに誤りがある場合は手動で修正してください。

2-2. メンバーブック(PDF)の更新

メニュー: 名簿システム > メンバーブック(PDF)の更新

メールでビジターに配布する「メンバーブック（資料）」のPDFをアップロードします。

  • 初回はGoogleドライブに新規作成、2回目以降は同じファイルを上書き更新します。
  • アップロード後、リンク付きで誰でも閲覧可能な状態に設定されます。
  • このPDFのURLはメールテンプレートの {{memberbook}} 変数で自動挿入されます。
  • AI自動割り振り時にも、メンバーブックの内容がGeminiに送信され、シナジー分析に活用されます。

2-3. 休会日の管理

メニュー: 名簿システム > 休会日の管理

お盆や年末年始など、定例会がお休みになる日を追加します。

  • 日付を入力して「追加」ボタンを押すと、リストに追加されます。
  • 不要な日付は「削除」ボタンで削除できます。
  • 登録された休会日は、定例会の候補日選択時に自動でスキップされます。
  • データは「休会日」シート（非表示）に保存されます。

2-4. メールテンプレート設定

メニュー: 名簿システム > メールテンプレート設定

各ビジターに送るメールの件名と本文を設定します。

  • 種別ごとのテンプレート: Visitor（ビジター）、Guest（ゲスト）、Substitute（代理）の3種別に、それぞれ件名・本文テンプレートを設定できます。
  • 共通CC/BCC: 全メールに共通のCC・BCCアドレスを設定できます。
  • 利用可能な差し込み変数:
変数 内容
{{name}} 参加者氏名
{{inviter}} 招待者氏名
{{date}} 開催日（YYYY年M月D日 形式）
{{visitorlist}} ビジター様リストPDFのURL
{{memberbook}} メンバーブックPDFのURL

2-5. 割り振り表の特記事項設定

メニュー: 名簿システム > 割り振り表の特記事項設定

割り振り表PDFの下部に記載する「当日の動き」などの注意書きを自由に編集できます。

  • 複数行入力が可能です。
  • 1行目に ※ をつけると太字・オレンジ色で強調表示されます。
  • 既定値として「ビジターホストの皆様へ（当日の動き）」のサンプルテキストが設定されています。

2-6. ビジターホスト・優先順位の設定

メニュー: 名簿システム > ビジターホストの設定

ビジターホストの指定と、オリエンテーション割り振りの優先順位を設定します。

  • ビジターホスト（チェックボックス）: チェックを入れたメンバーは、AI自動割り振り時に ファシリテーター および オリエンテーション担当 として優先配置されます。
  • 優先順位（数値入力）: 全メンバーに対して数値を設定できます。数値が 小さいほど優先度が高く、オリエンに優先的に割り振られます。空欄のメンバーは最も優先度が低い 扱いになります。
  • ビジターホストの中でも優先順位の数値が若い順にファシリ・オリエンへ配置されます。
  • 確度が高い順・Visitor優先でビジターをソートし、優先順位が若いホストから順番に割り当てられます。

2-7. Gemini API・モデル設定

メニュー: 名簿システム > Gemini API・モデル設定

割り振りの「AI自動化」を使うためのAPIキーを設定します。

  • APIキー: Google AI Studio から取得したGemini APIキーを入力してください。
  • モデル名: 使用するGeminiモデルを指定します（デフォルト: gemini-2.5-flash）。
  • APIキーはスクリプトプロパティに安全に保存されます。

2-8. Webアプリ（送信元）設定

メニュー: 名簿システム > Webアプリ(送信元)設定

メールの送信元アカウントを制御するための設定です。

  • WebアプリURL: GASをWebアプリとしてデプロイした際に取得するURLを入力します（詳細は後述の「Webアプリのデプロイ」を参照）。
  • 認証トークン: 送信元と受信元で共有する認証トークンです（デフォルト値が設定済み）。
  • URLが 空の場合: スプレッドシートを操作中のユーザーのGmailアカウントから直接メールが送信されます。
  • URLが 設定されている場合: Webアプリをデプロイしたユーザー（通常はチャプターの管理者アカウント）のGmailからメールが送信されます。

セキュリティ: WebアプリURLと認証トークンはスクリプトプロパティに保存されるため、ソースコード（Gitリポジトリ）には含まれません。

3. ビジター名簿とPDFの作成（毎週の作業）

Spreadingのデータから、見やすい参加者名簿シートとPDFを作成します。

メニュー: 名簿システム > 1. CSVから名簿・PDF作成

操作手順

  1. メニューから 「1. CSVから名簿・PDF作成」 をクリックします。
  2. 対象の定例会（日付と回数）をプルダウンから選びます。
  3. ダウンロードしておいた 「ビジターリストのCSV」 を点線枠内にドラッグ＆ドロップし、「全データを解析して編集画面へ」 を押します。
  4. データの全件確認・編集画面 が開きます。
  5. スペースが抜けている名前は自動で推測され 赤く 表示されます。
  6. 招待者がメンバー名と完全に一致しない場合は 黄色く 表示されます。
  7. 不自然な箇所があれば、Excelのように画面上で直接文字を打ち換えて修正してください。
  8. 「この内容でシートとPDFを作成する」 をクリックすると、スプレッドシートにデータが書き込まれ、同時に全員が閲覧可能なPDF（ビジター様リスト）が自動生成されます。

自動処理の詳細

処理 内容
Shift_JIS読み取り SpreadingのCSV（Shift_JISエンコーディング）を自動的に正しく読み取り
全角→半角変換 英数字の全角文字を自動的に半角に変換
名前スペース推測 姓名の間にスペースがない場合、3〜5文字なら2文字目以降にスペースを自動挿入
招待者マッチング メンバーリストと照合し、部分一致でも正式名称に自動補完
ソート 種別順（Visitor → Guest → Substitute）→ ふりがな順に自動整列
採番 Visitor=V01〜、Guest=G01〜、代理=代理+メンバーNo で自動付番

作成されるシート

  • MMdd参加者: 全データ（メールアドレス等を含む）の作業用シート
  • MMdd参加者_印刷用: PDF出力用の整形済みシート（7列: No., 参加者氏名, ふりがな, カテゴリー, 会社名, 招待者, 備考）

4. ルーム・オリエン割り振り表の作成（毎週の作業）

作成した名簿を基に、朝のブレイクアウトルームとオリエンテーションの担当を決めます。

メニュー: 名簿システム > 3. ルーム・オリエン割り振り表

操作手順

  1. メニューから 「3. ルーム・オリエン割り振り表」 をクリックします。
  2. 対象の定例会を選択して 「データを読み込む」 を押します。
  3. 画面が開き、以下の機能を使って割り振りを行います。

画面の機能詳細

ダッシュボード

画面上部に以下の情報がリアルタイムで表示されます。

  • ビジター・ゲスト・代理のそれぞれの人数
  • 現在のルーム数
  • 1ルームあたりの平均総人数（ビジター本人・招待者・ファシリ・ルームメンバーの合計。オリエンは含まない。同じ招待者が複数人を招待している場合、その招待者は 1/招待数 で按分カウントされる）
  • 未配置メンバーの残り人数

全情報ボタン（▼）

ビジターの確度、性格、メモなどをその場でパッと展開して確認できます。CSVに含まれる全カラムの詳細データを表示します。

つなげたいメンバー（手動入力）

SpreadingのCSVに載らない「つなげたいメンバー」は、Spreading画面を見ながらこの欄に改行区切りで入力（コピペ）してください。入力して枠外をクリックすると、自動的にその人がルームメンバーとしてチップで配置されます。

ドラッグ＆ドロップ（完全自由）

  • 画面下の待機リスト（プール）から、メンバーのチップをドラッグして配置します。
  • 配置済みのチップも直接ドラッグして、他のルームのファシリやルームメンバー、オリエン枠へ自由に移動（お引越し）させることができます。
  • チップには「No. 氏名」の形式でメンバー情報が表示されます。

合同ルームの設定

ビジター同士を同じ部屋にする場合、ファシリ枠の上のプルダウンから「合同先」を選びます。すると、設定されていたメンバーが自動的に親ルームへ引っ越し、設定が同期されます。

合同ルームのチェーン（A→B→C のような三重合同）は自動的に防止されます。

Gemini AI 自動割り振り

  1. 「最大人数（デフォルト5名）」 を指定して 「AIに提案させる」 ボタンを押します。
  2. AIが数分かけてPDFのメンバーブック情報やビジターのメモ（シナジー）を解析します。
  3. システム側で 「確度が高い順 ＞ Visitor優先」 でビジターホストをファシリテーター＆オリエンに優先配置します。
  4. その上で全ルームの人数が均等になるようにAIが残りのメンバーを最適配置します。

AIの割り振りロジック

  1. ビジターホストの優先配置: ビジターを確度順（高→低）→ 種別順（Visitor > Guest > Substitute）でソートし、優先順位の数値が若い順にビジターホストをファシリテーターへ順次割当
  2. ファシリ＝オリエン担当: ファシリテーターに配置されたメンバーは自動的にオリエン担当も兼務
  3. ルームメンバーのAI配置: Gemini APIがメンバーブックPDFの内容、ビジターのメモ、つなげたいメンバー情報を総合的に分析して最適配置
  4. 制約条件: 1人のメンバーは1ルームのみ、合同ルームには割当なし、最大人数制限を遵守

出力

「この内容でシートとPDFを作成する」 をクリックすると以下が作成されます。

  • MMdd割り振り表シート: 全割り振りデータ
  • 割り振り表PDF: A4縦向きで以下を含む
  • ダッシュボード（人数統計）
  • ビジターごとの割り振り一覧（ファシリテーター、ルームメンバー、オリエンテーション）
  • 特記事項（当日の動き）
  • メンバー別 逆引き担当表（全メンバーを掲載。ルームに入らないメンバーも名前が表示される）

5. メールの確認・一括送信

完成したPDFのURLを差し込んだ案内メールを一括送信します。

メニュー: 名簿システム > 2. メールの確認・一括送信

前提条件

先ほどの「名簿作成」で生成された 「〇〇参加者」というシート（タブ）を開いた状態 で実行してください。

操作手順

  1. メニューから 「2. メールの確認・一括送信」 をクリックします。
  2. 全員のメールのプレビュー画面（宛先、件名、本文）が表示されます。
  3. テンプレートの設定に基づき、氏名や開催日、ビジターリスト・メンバーブックのPDF URLがすでに差し込まれています。
  4. 各メールには種別（Visitor/Guest/Substitute）がバッジで表示されます。
  5. 個別に追記・修正したい文章があれば、この画面で直接テキストエリアを編集できます。
  6. 送信したくない人（既に個別連絡済みなど）がいる場合は、右上の 「送信する」のチェック を外します。
  7. 「上記の内容で順番に送信する」 をクリックすると、ゲージが進みながら1件ずつ確実に送信が行われます。

メール送信の仕組み

  • 送信者名は「Activeチャプター」として送信されます。
  • CC/BCCはテンプレート設定で指定したアドレスが全メールに適用されます（送信画面でも上書き可能）。
  • エラーがあった場合は画面に赤字で報告され、システムが途中で止まることはありません（残りのメールは引き続き送信されます）。
  • 無効なメールアドレス形式の場合はスキップされ、エラーとして報告されます。
  • 進捗はプログレスバーでリアルタイム表示されます。

メール送信元の制御（2つのモード）

本システムでは、メールの送信元を以下の2つのモードで切り替えられます。

モード 条件 送信元
直接送信モード WebアプリURLが未設定（空） スプレッドシートを操作中のユーザーのGmail
Webアプリ経由モード WebアプリURLが設定済み WebアプリをデプロイしたユーザーのGmail

チャプター共有のメールアドレス（例: 管理者のGmail）から送信したい場合は、そのアカウントでWebアプリをデプロイし、URLを設定してください。

5-2. ビジター情報サマリーの確認

定例会ごとのビジター・ゲスト・代理の参加者情報と入金状況を一覧表示し、テキストとしてコピーできる機能です。LINEグループやチャットへの共有に便利です。

メニュー: 名簿システム > 4. ビジター情報サマリー

操作手順

  1. メニューから 「4. ビジター情報サマリー」 をクリックします。
  2. 画面上部のプルダウンで 対象の定例会日付を選択 します。デフォルトでは次回（最新）の定例会が選択されています。
  3. 選択した定例会のビジター情報が自動的に読み込まれ、以下の内容がテキスト形式で表示されます。
    • 定例会の回数と日付
    • ビジター・ゲスト・代理それぞれの人数
    • 各参加者の氏名・ふりがな・カテゴリー・招待者
    • 入金済み / 未入金の区分（請求額と支払額の比較で自動判定）
  4. 「クリップボードにコピー」 ボタンをクリックすると、表示内容がそのままコピーされます。LINEやSlackなどに貼り付けて共有できます。

入金判定のしくみ

入金状況は、参加者データの 「請求額」列と「支払額」列 を比較して自動判定されます。

  • 未入金: 請求額が0より大きく、支払額が請求額未満の場合
  • 入金済み: 支払額が請求額以上の場合
  • 請求額の列がない場合は判定不能のため「入金済み」として扱われます

過去の定例会データの参照

プルダウンには、スプレッドシート内に存在するすべての MMdd参加者 シートが新しい日付順で表示されます。過去の定例会を選択すれば、その回のビジター情報を確認できます。

6. 作成済みPDFの再確認

「さっき作ったPDFのURLをもう一度確認したい」という場合に使います。

メニュー: 名簿システム > 4. 作成済みPDFの確認

操作手順

  1. メニューから 「4. 作成済みPDFの確認」 をクリックします。
  2. 直近で作成された以下のリンクボタンが表示されます。
  3. ビジター様リスト PDF
  4. ルーム割り振り表 PDF
  5. メンバーブック PDF
  6. クリックするだけで、Googleドライブ上の該当PDFをすぐに開くことができます。
  7. まだ作成されていないPDFはボタンがグレーアウト（無効化）されます。

7. Webアプリのデプロイ（送信元アカウントの設定）

特定のGoogleアカウントからメールを送信したい場合、GASプロジェクトを「Webアプリ」としてデプロイする必要があります。

なぜWebアプリのデプロイが必要か

スプレッドシートのスクリプトからメールを送信すると、操作しているユーザーのGmail から送信されます。しかし、チャプターの運営では「誰が操作しても同じ送信元（管理者のGmail）から送信したい」というケースがあります。

Webアプリとしてデプロイすると、デプロイしたユーザーの権限 でスクリプトが実行されるため、送信元を固定できます。

デプロイ手順

  1. 送信元にしたいGoogleアカウント でGASエディタを開きます。
  2. スプレッドシートのメニューから 拡張機能 > Apps Script をクリック。
  3. 右上の 「デプロイ」 ボタン > 「新しいデプロイ」 をクリックします。
  4. 左上の歯車アイコンをクリックし、種類として 「ウェブアプリ」 を選択します。
  5. 以下の設定を行います。
設定項目 値

デプロイの更新（コード変更時に必須）

重要: GASエディタやclasp pushでコードを変更しただけでは、Webアプリには反映されません。コードを変更するたびに、以下の手順でデプロイを更新する必要があります。Webアプリは「デプロイ時点のコードのスナップショット」で動作しているためです。

GASエディタから更新する場合

  1. GASエディタで 「デプロイ」 > 「デプロイを管理」 をクリックします。
  2. 鉛筆アイコン（編集）をクリックします。
  3. バージョンを 「新しいバージョン」 に変更します。
  4. 「デプロイ」 をクリックします。

注意: 「新しいデプロイ」ではなく「デプロイを管理」から更新してください。新しいデプロイを作成するとURLが変わり、設定の再登録が必要になります。

clasp から更新する場合

# 1. コードをプッシュ
clasp push

# 2. 既存のデプロイIDを確認
clasp deployments

# 3. デプロイを更新（URLが変わらない）
clasp deploy -i <デプロイID> --description "更新内容のメモ"

コード変更後の作業まとめ

手順 コマンド / 操作 説明
1 clasp push または GASエディタで編集 コードをGASプロジェクトに反映
2 clasp deploy -i <ID> または「デプロイを管理」から更新 Webアプリに最新コードを反映

clasp push だけで終わると、スプレッドシートから直接実行する機能（メニュー操作など）は更新されますが、Webアプリ経由のメール送信は古いコードのまま になります。

clasp を使ったデプロイ

コマンドラインからデプロイする場合:

# 初回デプロイ
clasp deploy --description "メール送信用"

# デプロイ後、表示されるデプロイIDを確認
clasp deployments

# コード更新後の再デプロイ（既存のデプロイIDを指定）
clasp deploy -i <デプロイID> --description "メール送信用 v2"

デプロイ後のWebアプリURLは以下の形式です:

https://script.google.com/macros/s/<デプロイID>/exec

定例会の基準日計算

定例会は毎週水曜日に開催されます。候補日の算出は以下のロジックで行われます。

  • 基準日: 2026/03/18（第509回）
  • 計算: 基準日から7日ずつ加算し、今日以降かつ休会日でない直近4回分を候補として表示
  • 回数: 休会日でない場合のみカウントアップ（休会日は回数をスキップ）

よくあるトラブルと対処法

Q: メニューに「名簿システム」が表示されない

スプレッドシートを開き直してください。onOpen() 関数がメニューを生成するため、初回読み込み時にメニューが作成されます。

Q: CSVのドラッグ＆ドロップが反応しない

Spreadingからダウンロードしたそのままのファイル（Shift_JIS形式）を使用してください。テキストエディタ等で別のエンコーディングに変換した場合、文字化けの原因になります。

Q: メンバーリストのOCR結果がおかしい

PDFの画質やフォーマットによっては正しく読み取れない場合があります。「メンバーリスト」シートを直接手動で修正してください。

Q: AI割り振りでエラーが出る

  • Gemini APIキーが正しく設定されているか確認してください。
  • メンバーブックPDFがアップロードされているか確認してください（なくても動作しますが、精度が下がります）。
  • APIの利用制限に達していないか確認してください。

Q: メール送信でエラーが出る

  • 「〇〇参加者」シートがアクティブ（表示中）になっているか確認してください。
  • メールアドレスに @ が含まれていない場合は無効と判定されスキップされます。
  • Gmailの1日あたりの送信制限（通常500件/日）を超えていないか確認してください。
